Kafka is a distributed message streaming platform and one of the Apache projects. It can be use for numerous tasks: processing queue, messaging system, storage, etc. It can be use to build real-time processing system. It is horizontally scalable, fault-tolerant and quite fast.

Amazon has its own implementation streaming platform similar to Kafka called Kinesis. TODO: Kinesis description

However, you might prefer Kafka over Kinesis but you will have to deploy it yourself on top of EC2 instances. Using EC2, you might want to use CloudWatch to monitore your instance. CPU, disk and network usage metrics are recorded out of the box and you can even expend it with the CloudWatch agent to gather memory usage metrics. But when you operate a Kafka broker, a very important metric to track to keep an eye on the processing flow health is the consumer lag.
